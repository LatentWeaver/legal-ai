# Legal-AI: Citation Network Analysis of Indian Supreme Court Land & Property Dispute Cases

## Overview

This project builds a **directed citation graph** from 7,496 Indian Supreme Court land and property dispute judgments (1950–2024). Case metadata is sourced from [Indian Kanoon](https://indiankanoon.org). Citations between cases are extracted via automated browser scraping to construct a network that reveals how judicial precedent flows across decades of property law.

The full corpus is split across team members, each responsible for a window of cases. Results from each window are stored in a dedicated `data_<start>_<end>/` folder and can be merged for full-corpus analysis.

## Output File Format

| File | Columns | Description |
|---|---|---|
| `nodes.csv` | `id, case, year` | One row per case in the full corpus |
| `edges.csv` | `citing_id, cited_id` | Intra-corpus citation edges only |
| `out_citations_raw.csv` | `citing_id, cited_id` | All observed citations including external targets |

## How to Reproduce the Scraping

### 1. Install dependencies

```bash
pip install requests beautifulsoup4 networkx matplotlib playwright playwright-stealth
python -m playwright install chromium
```

> **Chrome required**: the scraper uses your locally installed Google Chrome (`channel="chrome"`) to bypass Cloudflare bot protection. Make sure Chrome is installed.

### 2. Run the extractor for a window of cases

The full corpus has 7,496 cases (0-based indices 0–7495). Each team member runs a slice using `--start` and `--limit`.

```bash
python extract_citations.py \
  --start 2250 \
  --limit 750 \
  --outdir data_2251_3000 \
  --cachedir cache_2251_3000 \
  --workers 1 \
  --rate 3.0 \
  --browser
```

| Flag | Meaning |
|---|---|
| `--start N` | 0-based index of the first case to process |
| `--limit N` | Number of cases to process (so `--start 2250 --limit 750` covers cases 2251–3000) |
| `--outdir DIR` | Where to write `nodes.csv`, `edges.csv`, `out_citations_raw.csv` |
| `--cachedir DIR` | Where to store raw HTML cache (gitignored, safe to delete) |
| `--workers N` | Parallel fetch workers — **keep at 1** when using `--browser` |
| `--rate N` | Minimum seconds between requests (3.0 recommended) |
| `--browser` | Use a live Playwright Chrome session (handles Cloudflare automatically) |
| `--headless` | Hide the browser window (not recommended — Cloudflare detects headless more aggressively) |

### 3. Cloudflare checkpoint

When the browser opens for the first time, Cloudflare may show a **"Verify you are human"** checkbox. Click it once. The script logs `Challenge solved, resuming.` and continues automatically for the rest of the run. You will not need to click again during the same session.

If the script is interrupted and restarted, already-fetched pages are read from the cache — only new pages are fetched from the web.

### 4. Typical run time

At `--rate 3.0` with a single worker, 750 cases takes approximately **60–90 minutes** depending on how many pages of results each case has.

## Build the Citation Graph

After extraction, build the graph and generate stats + visualization:

```bash
python build_graph.py --outdir data_2251_3000
```

This reads `nodes.csv` and `edges.csv` from `--outdir` and writes `report.md` and `degree_dist.png` into the same folder.
